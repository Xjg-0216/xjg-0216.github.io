    Conda 命令¶

    基础命令¶

    1.查看Conda版本¶

    Bash
    1
    conda --version

    2.更新Conda¶

    Bash
    1
    conda update conda

    3.获取Conda帮助¶

    Bash
    1
    conda --help

    环境管理¶

    1.创建新环境¶

    创建新环境并指定Python版本

    Bash
    1
    conda create --name myenv python=3.8

    2.克隆环境¶

    Bash
    1
    conda create --name myenv_clone --clone myenv

    3.激活环境¶

    Bash
    1
    conda activate myenv

    4.停用环境¶

    Bash
    1
    conda deactivate

    5.列出所有环境¶

    Bash
    1
2
3
    conda env list
# 或
conda info --envs

    6.删除环境¶

    Bash
    1
    conda remove --name myenv --all

    包管理¶

    1.安装包¶

    Bash
    1
    conda install numpy

    2.安装特定版本的包¶

    Bash
    1
    conda install numpy=1.18.1

    3.从特定的频道安装包¶

    Bash
    1
    conda install -c conda-forge numpy

    4.更新包¶

    Bash
    1
    conda update numpy

    5.更新所有包¶

    Bash
    1
    conda update --all

    6.删除包¶

    Bash
    1
    conda remove numpy

    7.强制删除包¶

    Bash
    1
    conda remove numpy --force

    8.查看已安装的包¶

    Bash
    1
    conda list

    9.查看某个包的详细信息及依赖¶

    Bash
    1
    conda info numpy

    环境导出与导入¶

    1.导出环境到YAML文件¶

    Bash
    1
    conda env export > environment.yml

    2.从YAML文件创建环境¶

    Bash
    1
    conda env create -f environment.yml

    3.导出环境到简单列表¶

    Bash
    1
    conda list --export > requirements.txt

    4.从简单列表创建环境¶

    Bash
    1
    conda create --name myenv --file requirements.txt

    清理缓存¶

    1.清理已下载的包文件¶

    Bash
    1
    conda clean --packages

    2.清理所有缓存¶

    Bash
    1
    conda clean --all

    Conda 配置管理¶

    Conda频道是包集合的源，Conda通过这些渠道来查找、下载和安装软件包。不同的频道可能包含不同版本的软件包，或者有不同的维护和更新策略。

    常见的Conda频道 默认频道（defaults）:
    这是Conda的官方频道，默认情况下所有Conda安装都使用这个频道。它包含了大量常用的包，维护和更新由Anaconda团队负责。 URL: https://repo.anaconda.com/pkgs/main/
    conda-forge:
    这是一个社区驱动的频道，包含了更多包和更新版本，通常比默认频道更新更快。
    URL: https://conda-forge.org/
    r:
    专门提供 R 语言相关的包。
    URL: https://repo.anaconda.com/pkgs/r/
    free:
    包含了所有许可证下的自由和开放源码软件包，这是默认频道的一部分，但在Anaconda 5.0之后被合并到了 main 频道中。
    URL: https://repo.anaconda.com/pkgs/free/

    1.添加频道¶

    Bash
    1
    conda config --add channels conda-forge

    2.添加频道并置于最高优先级¶

    Bash
    1
    conda config --add channels conda-forge --prepend

    3.添加其他频道¶

    Bash
    1
    conda config --add channels defaults

    4.移除频道¶

    Bash
    1
    conda config --remove channels conda-forge

    5.删除所有频道¶

    Bash
    1
    conda config --remove-key channels

    6.验证频道¶

    Bash
    1
    conda config --get channels

    7.设置HTTP代理¶

    Bash
    1
    conda config --set proxy_servers.http http://proxy_address:port

    8.设置HTTPS代理¶

    Bash
    1
    conda config --set proxy_servers.https https://proxy_address:port

    9.设置环境的默认目录¶

    Bash
    1
    conda config --set envs_dirs /path/to/envs

    10.启用详细输出¶

    Bash
    1
    conda config --set verbosity 2

    11.禁用包版本检查¶

    Bash
    1
    conda config --set auto_update_conda false

    12.列出所有配置¶

    Bash
    1
    conda config --show

    13.查看包信息¶

    Bash
    1
    conda search numpy --info

    14.查看可用包¶

    Bash
    1
    conda search numpy

    使用conda-pack打包¶

    1. 打包 Conda 环境¶

    首先，在有网络的环境下，使用 conda-pack 对环境进行打包：

    1. 安装 conda-pack：

      Bash
      1
      conda install -c conda-forge conda-pack

    2. 打包当前的 Conda 环境：

      Bash
      1
      conda-pack -n my_env -o my_env.tar.gz
      这里 my_env 是要打包的 Conda 环境的名称，my_env.tar.gz 是打包生成的压缩文件名。

    2. 将打包文件传输到无网络的环境¶

    将生成的 my_env.tar.gz 文件传输到无网络的目标机器上。

    3. 在无网络环境下解压并安装¶

    在无网络环境的机器上进行以下操作：

    1. 解压缩打包文件：

      Bash
      1
2
      mkdir -p ~/my_env 
tar -xzf my_env.tar.gz -C ~/my_env

    2. （可选）如果你希望将解压后的环境放置在特定路径下（例如 /opt/conda/envs/my_env），你可以将解压后的内容移至该路径：

      Bash
      1
      mv ~/my_env /opt/conda/envs/my_env

    3. 激活解压后的环境：

    Bash
    1
    source ~/my_env/bin/activate
    或者，如果你移动了环境目录：
    Bash
    1
    source /opt/conda/envs/my_env/bin/activate

    1. 修复环境路径（如果需要）： 如果移动了解压后的环境到一个新的路径，你需要修复环境中的路径信息：
      Bash
      1
      conda-unpack

    这一步会修复环境中与原路径相关的依赖文件路径，使环境能够正常运行。

    4. 验证安装¶

    最后，验证环境是否正确安装并可以正常使用：

    Bash
    1
    conda list

    这将显示当前环境中的所有已安装包。

    通过以上步骤，你可以在无网络环境下成功安装和使用一个已打包的 Conda 环境。
