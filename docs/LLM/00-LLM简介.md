# LLM基本概念

## Agent


**智能体（Agent）是一种能够自主调用工具、检索信息、规划任务并执行动作的大模型应用形式。**

它不仅回答问题，还**能主动思考“我应该做什么”，并使用外部工具或执行代码完成任务**，像一个有“执行能力”的 AI 助手。

**一个智能体的组成**

```
用户输入
   ↓
智能体思考（大模型）
   ↓
决定行动（例如检索 / 计算 / 调工具）
   ↓
执行动作
   ↓
将结果返回给模型继续思考
   ↓
最终输出结果
```

---
**举例**

用 **“出行规划”** 举一个智能体（Agent）完整工作流程的例子

> “我下周一从深圳出发去北京出差，要 10 点前到客户公司，帮我规划出行方案，包含交通方式、费用和时间，并且提醒我带上必要证件。”


### 智能体执行过程（内部思考+行动）

#### 第1步：理解目标

* 出发地：深圳
* 到达地：北京
* 时间：下周一上午 10 点前
* 目的：客户公司
* 附加要求：费用 + 时间安排 + 必带东西

#### 第2步：规划任务步骤（自主生成）

1. 查询深圳→北京交通方式（工具1：地图或出行API）
2. 比较飞机、高铁费用和时间
3. 查询天气（工具2）
4. 结合天气推荐出行方式
5. 生成时间表（模型规划）
6. 提醒要带什么（模型生成）
7. 最终返回方案

#### 第3步：依次调用工具并分析结果

* **调用出行工具**：查询航班、高铁数据
* **调用天气工具**：查看当天是否有雨雪
* **模型分析**：如果有雪 → 高铁更稳妥
* **计算总时间**：含提前2h值机/车站排队
* **模型分析公司位置 → 路程时间**

#### 第4步：生成最终计划


> **智能体（Agent）能像真人助理一样 → 先分析需求 → 决定做什么 → 调用工具 → 分析结果 → 给你最佳方案。
> 它不是工具调用器，而是“能帮你完成任务的 AI”**。
